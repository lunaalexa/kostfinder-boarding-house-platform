# KostFinder 🏠

KostFinder is a boarding house search and comparison platform designed to help students and young professionals find accommodations more efficiently. The project was developed as part of a Human-Computer Interaction (HCI) group project and focuses on improving the user experience of searching, comparing, and booking boarding houses (kos).

The idea came from a simple frustration we experienced ourselves: opening multiple tabs just to compare different boarding houses.

## ✨ Project Overview

**Problem**

Many existing boarding house platforms provide limited filtering options and make it difficult to compare multiple accommodations side-by-side. As a result, users often spend significant time switching between listings before making a decision.

**Solution**

KostFinder introduces a dedicated comparison feature that allows users to evaluate multiple boarding houses in one place, making the search process faster and more informed.

**Project Status**

This project was developed as a prototype and concept validation project. It was not deployed as a production system.

## 🔍 Development Process

* Conducted user research to identify common pain points in the boarding house search process
* Designed user flows and interactive prototypes in Figma
* Developed a multi-page web prototype using HTML, CSS, JavaScript, and localStorage
* Performed usability testing and evaluation using SUS and UEQ

## 📊 Key Findings

* Several features that initially seemed intuitive to the design team were not immediately understood by users
* Iterative redesigns significantly improved usability and user satisfaction
* The final prototype achieved a SUS score of **79.5 (Good)**

---

## 💡 What I Learned

This project taught me that what feels intuitive to designers is not always intuitive to users. Continuous testing and iteration were far more valuable than trying to perfect the first design.
